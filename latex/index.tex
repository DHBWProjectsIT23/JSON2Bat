JSON2\+Batch was developed during a project during our first and second semester of university. It generates batch files from JSON files, which can spawn terminals or applications, that run under certain parameters specified within the JSON file.

\hypertarget{index_autotoc_md11}{}\doxysubsection{\texorpdfstring{easylogging++}{easylogging++}}\label{index_autotoc_md11}
The \href{https://github.com/abumq/easyloggingpp}{\texttt{ easylogging++}} library is used for logging within the application. The configuration for the library is done via a logging file which can be found in \href{conf/easylogging.in.conf}{\texttt{ conf/easylogging.\+in.\+conf}}. Cmake configures this file into the binary directory upon building. If the configuration file is removed, the application will no longer run.\hypertarget{index_autotoc_md12}{}\doxysubsection{\texorpdfstring{Logging\+Wrapper}{LoggingWrapper}}\label{index_autotoc_md12}
While easylogging++ is used for the logging back--end within the code there are little remains apart from the configuration. The logging and output of the application is done over a self written wrapper. Altough it is self written, due to it beeing not part of the project we consider it an external libraries. The wrapper is used to simplify parallel output to stdout and the logfile and also enables increased output to stdout for the verbose mode. A few macros are defined for use within the application\+:


\begin{DoxyItemize}
\item OUTPUT \texorpdfstring{$>$}{>} {\itshape Outputs to stdout and the logfile}
\item LOG\+\_\+\+INFO \texorpdfstring{$>$}{>} {\itshape By default only outputs to the logfile}
\item LOG\+\_\+\+WARNING \texorpdfstring{$>$}{>} {\itshape Formats text and outputs to stdout and the logfile}
\item LOG\+\_\+\+ERROR \texorpdfstring{$>$}{>} {\itshape Same as LOG\+\_\+\+WARNING but in red and bold}
\end{DoxyItemize}

The macros can be used with streaming in the same way as std\+::cout would be used. Furthermore, some rudimentary performance tests showed, that the use of the wrapper, does not affect performance in comparison to using both std\+::cout and easylogging itself.\hypertarget{index_autotoc_md13}{}\doxysubsection{\texorpdfstring{jsoncpp}{jsoncpp}}\label{index_autotoc_md13}
For parsing the JSON files, the \href{https://github.com/open-source-parsers/jsoncpp}{\texttt{ jsoncpp}} library is used. On UNIX system this library can simply be installed using the systems package manager (tested with WSL/\+Ubuntu and Arch).
